# ユースケース

## アクター

| アクター | 概要 |
|---|---|
| 部長・マネージャー | メンバー、案件、評価、推薦業務を担当する |
| システム運用者 | システム設定、AI設定、監査、物理削除を担当する |

## UC-01 メンバー登録

- アクター: 部長・マネージャー
- 目的: 推薦対象となるメンバーを登録する
- 事前条件: 操作者が対象部署を管理できる
- 基本フロー:
  1. メンバー一覧を開く
  2. 氏名、部署、管理者を入力する
  3. 保存する
  4. 監査ログを記録する

## UC-02 案件経験登録

- アクター: 部長・マネージャー
- 目的: メンバー単位の案件経験を登録する
- 基本フロー:
  1. メンバー詳細を開く
  2. 案件経験を追加する
  3. 案件名、期間、概要などを入力する
  4. 保存する

## UC-03 Markdown報告取り込み

- アクター: 部長・マネージャー
- 目的: メンバーが作成した案件報告を取り込む
- 基本フロー:
  1. メンバーを選択する
  2. 案件を選択する
  3. Markdownファイルを指定する
  4. AI Gateway経由で解析する
  5. 解析できた項目を自動保存する
  6. スキル候補を保存する
  7. 曖昧な項目を警告表示する
  8. 必要に応じて上司が修正する

## UC-04 案件報告登録・更新

- アクター: 部長・マネージャー
- 目的: 定期報告、終了報告、任意報告を登録・更新する
- 拡張: 上司は任意でAI案件分析を実行できる

## UC-05 人物評価登録

- アクター: 部長・マネージャー
- 目的: 時系列の人物評価を登録する
- 評価種別:
  - 定期評価
  - 案件終了評価
  - 任意評価

## UC-06 推薦プロジェクト作成

- アクター: 部長・マネージャー
- 目的: 推薦文作成作業をプロジェクトとして開始する
- 入力:
  - 推薦対象
  - 推薦目的
  - 推薦先
  - 要件
  - 強調点
  - 文体
  - 出力形式

## UC-07 推薦文生成

- アクター: 部長・マネージャー
- 目的: 根拠付き推薦文ドラフトを生成する
- 基本フロー:
  1. 最新情報を収集する
  2. 推薦目的に適した根拠を抽出する
  3. 推薦文ドラフトを生成する
  4. 段落と根拠を紐付ける
  5. 根拠不足を警告する
  6. バージョンとして保存する
- 制約:
  - 推薦可否を判断しない
  - スコアリングしない
  - 根拠のない内容を書かない

## UC-08 推薦文レビュー・修正

- アクター: 部長・マネージャー
- 目的: AIドラフトを上司の判断で修正する
- 基本フロー:
  1. ドラフトと根拠を確認する
  2. 文章を編集する
  3. 新しいバージョンを保存する
  4. 必要に応じて再生成する

## UC-09 推薦文確定

- アクター: 部長・マネージャー
- 目的: 最終版を確定する
- 結果: 推薦プロジェクトを完了状態にする

## UC-10 AIモデル設定

- アクター: システム運用者
- 目的: AI Gateway接続およびモデルを設定する
- 管理項目:
  - Base URL
  - APIキー
  - モデル名
  - タイムアウト
  - 最大トークン
  - 温度
  - 用途別モデル
  - プロンプトバージョン

## UC-11 監査ログ確認

- アクター: 部長・マネージャー、システム運用者
- 目的: 許可された範囲の変更履歴を確認する

## UC-12 データ復元・物理削除

- 復元: システム運用者
- 物理削除: システム運用者のみ
- 物理削除前に関連データと影響範囲を表示する

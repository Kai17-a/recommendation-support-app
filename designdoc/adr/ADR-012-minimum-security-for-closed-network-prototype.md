# ADR-012: 閉域プロトタイプではセキュリティ要件を必要最低限とする

## 状態

採用

## 決定

初期版は閉じた社内ネットワークで利用し、インターネットへ公開しない。

接続情報と秘密値は `.env` で管理し、`.env` はGitへコミットしない。企業IdP、Vault、Secret Manager、WAF、高可用性構成などは初期版の必須要件としない。

## 理由

今回の目的は推薦業務とAI支援機能のプロトタイプ検証であり、公開サービス相当のセキュリティ要件は実装負荷が過大である。

## 影響

- 実装を簡素化できる
- プロトタイプ作成を優先できる
- 外部公開や本番利用前にはセキュリティ再設計が必要
- `.env` の誤コミット防止は必須
